# metal/ — on-device (Metal + CoreML) inference

**Status: skeleton.** API surfaces, host wiring, and CPU post-processing are written;
the GPU kernel bodies and env sampling are stubbed (`TODO`). Build and finish this on a
Mac. Nothing here runs on Linux (Metal + CoreML are Apple-only) — but the parity target
in `parity/golden.json` is generated by the Python path and is platform-independent.

## The idea

The Python pipeline is `parse → descriptors → set-transformer → decode`. Only the
descriptor stage is expensive at scale, and it's plain histogram math — a perfect fit for
a GPU kernel. `InaccurateRDFCalculator` computes the per-type-pair descriptors on Metal
and hands them to the CoreML-exported network (`env_codfull_sharp30.mlpackage`, 3.6 MB
fp16; committed int8 ONNX is for the desktop lite path).

"Inaccurate" is a deliberate design license: chemistry is decided by bond-length **peaks**
and **coordination**, both robust to subsampling RDF centers or using a coarse neighbor
search. So the on-device descriptor may approximate `g_ab(r)` — parity is defined by
*predictions*, not bit-exact histograms.

## Descriptor contract (keep in sync with `src/typeid2elem/descriptors.py`)

| output | shape | notes |
|---|---|---|
| `rdf` | (T,T,64) | smeared, ideal-gas-normalized `g_ab(r)`, ordered (a←b), clip ≤1e4 |
| `pair_extra` | (T,T,8) | `[log1p(cn@2,3,4,6), nn_median/8, nn_p10/8, peak_pos, log1p(peak_h)]` |
| `frac` | (T,) | stoichiometric fractions |
| `glob` | (3,) | `[log(N/V), has_cell, 1/T]` — **no temperature/metadata, ever** |
| `env_d`,`env_t` | (T,16,16) | per type, 16 sampled atoms × 16 nearest (distance, partner type); −1 pad |

CoreML I/O (from `scripts/export_model.py`): inputs `rdf, pair_extra, frac, glob, mask,
env_d, env_t` (all padded to fixed **T=8**, `mask`/`env_t` are floats), output `log_probs`
(1,T,94). Real types fill the first `nTypes` rows; the rest are masked padding.

## Parity protocol

```bash
cd metal
swift run parity-check ../metal/parity/golden.json           # descriptors only
swift run parity-check ../metal/parity/golden.json path/to/env_codfull_sharp30.mlpackage
```

Pass criteria (loose by design):

| field | tolerance | why |
|---|---|---|
| `frac`, `glob` | 1e-4 / 1e-3 | exact counting math |
| `pair_extra` | 5e-3 | robust stats; peak/CN should be near-exact |
| `rdf` | 5e-2 max\|Δ\| | smearing/normalization; approximation budget lives here |
| top-1 per type | must match | the real acceptance test |

If top-1 and the 90% conformal set match the reference, the port is good even if the raw
RDF differs — that's the whole point of *Inaccurate*.

## What's left (in order)

1. **envSample kernel** + host bucketing (per-type atom lists via prefix sum), then
   4-draw pooling in `TypeSHI.predict` (mirror the Python inference default).
2. **Cell list / uniform grid** to replace the O(N²) inner loop in `rdfHistogram`; the
   interior-crop trick (`descriptors.crop_for_inference`) maps directly to grid cells and
   is how million-atom files stay cheap.

   pipeline, wrong for `nn_p10` under contention).
